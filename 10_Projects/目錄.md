# 10_Projects 目錄

## 資料夾用途

保存有明確成果、時程、完成條件的活躍設備軟體專案。資料夾採「機種 / 工單」兩層結構：機種資料夾集中共用資訊，工單資料夾追蹤實際交付。

## 建議結構

```text
10_Projects/
  <machine-model>/
    目錄.md
    model.md
    common-resources.md
    <work-order-id>/
      project.md
      schedule.md
      issues.md
      tasks.md
      decisions.md
      meetings.md
      resources.md
      outputs/
```

## 瀏覽重點

- 先看本頁 dashboard，掌握所有機種與工單的目前階段、下一步與阻塞。
- 需要看同機種多工單比較時，進入 `<machine-model>/目錄.md`。
- 需要單張工單細節時，讀該工單的 `project.md`、`schedule.md`、`issues.md`。
- 完成、暫停或不再活躍的工單，整包移到 `40_Archives/`，並更新本目錄與機種目錄。

## 設備專案標準階段

```text
開工 > 開 BOM > 組裝配線 > 送電 > IO Check > 機台測試 > 交機 move in > 客戶端組裝送電 > 整合測試 > release 產線
```

## 提醒追蹤規則

- 每張工單的 `schedule.md` 必須維護 `Target Date`、`Remind On`、`Owner`、`Status`。
- 每個影響時程的問題必須進入該工單的 `issues.md`，並連到對應 stage。
- 每次更新工單時，同步更新機種資料夾 `目錄.md` 與本 dashboard 的「目前階段」、「下一步」、「風險/阻塞」、「更新日期」。
- AI 不應只靠人工閱讀判斷提醒；處理專案管理問題時，必須執行 `toolbox/project_reminder_scan.py` 讓 Python 掃描到期與逾期項目。
- 若掃描結果顯示 `Remind On` 已到期或 `Target Date` 逾期且狀態不是 `done`，AI 應提醒使用者確認進度或建立 issue。

## 專案建立方式

新增機種時，複製：

```text
templates/equipment-model-template/
```

到：

```text
10_Projects/<machine-model>/
```

新增工單時，複製：

```text
templates/equipment-project-template/
```

到：

```text
10_Projects/<machine-model>/<work-order-id>/
```

然後更新工單 `project.md`、`schedule.md`、`issues.md`、機種 `目錄.md` 與本目錄。

## AI 檢索提示

AI 處理專案管理問題時，固定流程是：

1. 先讀本目錄確認活躍機種、工單與狀態。
2. 執行 `toolbox/project_reminder_scan.py`，用 Python 掃描提醒與逾期。
3. 若掃描結果涉及特定機種或工單，先讀 `<machine-model>/目錄.md`。
4. 若問題涉及時程，讀工單的 `schedule.md`。
5. 若涉及異常、風險、阻塞，讀工單的 `issues.md`。
6. 若涉及原因、取捨或歷史背景，再讀 `decisions.md`、`meetings.md`、`resources.md` 或機種 `common-resources.md`。
