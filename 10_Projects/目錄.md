# 10_Projects 目錄

## 資料夾用途

保存有明確成果、時程、完成條件的活躍專案。設備軟體專案應以資料夾為單位管理，包含專案總覽、裝機時程、任務、Issue list、決策、會議與資源。

## 瀏覽重點

- 先看本頁 dashboard，掌握每個專案的目前階段、下一步與阻塞。
- 需要細節時，再進入個別專案的 `project.md`、`schedule.md`、`issues.md`。
- 完成、暫停或不再活躍的專案，整包移到 `40_Archives/`，並更新本目錄。

## 設備專案標準階段

```text
開工 > 開 BOM > 組裝配線 > 送電 > IO Check > 機台測試 > 交機 move in > 客戶端組裝送電 > 整合測試 > release 產線
```

## 提醒追蹤規則

- 每個專案的 `schedule.md` 必須維護 `Target Date`、`Remind On`、`Owner`、`Status`。
- 每個影響時程的問題必須進入 `issues.md`，並連到對應 stage。
- 每次更新專案時，同步更新本 dashboard 的「目前階段」、「下一步」、「風險/阻塞」與「更新日期」。
- AI 不應只靠人工閱讀判斷提醒；處理專案管理問題時，必須執行 `toolbox/project_reminder_scan.py` 讓 Python 掃描到期與逾期項目。
- 若掃描結果顯示 `Remind On` 已到期或 `Target Date` 逾期且狀態不是 `done`，AI 應提醒使用者確認進度或建立 issue。

## 專案建立方式

新增設備軟體專案時，複製：

```text
templates/equipment-project-template/
```

到：

```text
10_Projects/<project-slug>/
```

然後更新 `project.md`、`schedule.md`、`issues.md` 與本目錄。

## AI 檢索提示

AI 處理專案管理問題時，固定流程是：

1. 先讀本目錄確認活躍專案與狀態。
2. 執行 `toolbox/project_reminder_scan.py`，用 Python 掃描提醒與逾期。
3. 若掃描結果涉及特定專案或 stage，再讀該專案的 `schedule.md`。
4. 若涉及異常、風險、阻塞，再讀 `issues.md`。
5. 若涉及原因、取捨或歷史背景，再讀 `decisions.md`、`meetings.md` 或 `resources.md`。
